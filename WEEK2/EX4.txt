1. Entity (JPA part)
import jakarta.persistence.*;

@Entity
@Table(name = "books")
public class Book {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String title;

    // getters and setters
}

👉 This uses JPA annotations (@Entity, @Id, @Table)

2. Repository (Spring Data JPA part)
import org.springframework.data.jpa.repository.JpaRepository;

public interface BookRepository extends JpaRepository<Book, Long> {
}

👉 No implementation needed
👉 Spring Data JPA auto-generates code

3. Service (uses repository)
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;

@Service
public class BookService {

    @Autowired
    private BookRepository repo;

    public void saveBook() {
        Book b = new Book();
        b.setTitle("Java Basics");
        repo.save(b);   // Hibernate actually executes SQL
    }
}
🔥 What is happening behind the scenes?
JPA → defines @Entity, @Id
Spring Data JPA → creates BookRepository automatically
Hibernate → actually runs SQL like:
INSERT INTO books (title) VALUES ('Java Basics');
🧠 One-line flow:

Spring Data JPA → JPA → Hibernate → Database
